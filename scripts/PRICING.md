# 印刷費・販売価格 計算アルゴリズム

`validate.py` の価格計算ロジックと、その背景にある判断基準を記録する。

---

## 1. 印刷費テーブルについて

### なぜ日光企画か

- 公式サイトに静的な料金テーブルが存在する（スクレイピング・手動参照が容易）
- 同人誌印刷所の中で技術書典サークルの利用率が高い
- オンデマンド印刷に対応しており、完全受注生産と相性が良い

### なぜ「参考値」か

印刷所はAPIを公開していない。価格は年に1〜2回改定される（シメケンは2025年5月に改定予定）。
ツールが出した数字を鵜呑みにして損をするリスクを避けるため、常に警告ヘッダーを付けて表示する。
入稿前は必ず公式サイトの見積もりフォームで最新価格を確認すること。

### テーブルの対象条件

- 印刷所: 日光企画
- 形式: オンデマンド印刷
- サイズ: A5
- 本文: モノクロ
- データ取得: 2025年1月

他の条件（カラー表紙・B5・オフセット）に変更した場合はテーブルを差し替える必要がある。

## 2. 完全受注生産の価格設定ロジック

### 完全受注生産とは

イベント当日に注文を受け、後日印刷して発送する方式。
在庫リスクがない代わりに、最小ロット（10部）単位で印刷するため1冊あたりの印刷費が高くなる。

### なぜ10部ロットを基準にするか

日光企画のオンデマンド印刷は10部が最小ロット。
完全受注生産では「その日の注文数 ≈ 10部」を最小単位として考えるのが現実的。
1冊注文でも10部注文でも同じロットで刷るため、単価は10部時の値を使う。

### 印刷費回収ラインの計算

```
印刷費回収ライン = ceil(1冊印刷費 × 2 / 100) × 100  （100円単位切り上げ）
```

**×2の根拠**: 印刷費のみでは搬入・搬出・梱包材・送料・イベント参加費・時間コストを賄えない。
同人誌の慣例として「印刷費の2倍を最低ラインとする」考え方が広く使われている。
これは赤字を出さない最低ラインであり、利益を出すための価格ではない。

### 市場標準との比較

60pの場合: 印刷費550円 → 回収ライン1,100円 vs 市場標準1,000円

回収ラインが市場標準を上回る場合は、以下の選択肢がある。
1. 頒布価格を回収ライン以上に設定する（市場標準より高い価格で売る）
2. まとめ刷りに切り替えて単価を下げる（20部なら385円/冊 → 市場標準で+12,300円の利益）
3. 赤字を許容して市場標準に合わせる（次回作・認知度向上への投資として）

## 3. PDF価格の逆算ロジック

### なぜ物理本から逆算するか

PDF価格を「市場相場から直接決める」のではなく、「物理本価格を先に決めてから相対的に設定する」方が
購買者に価格の正当性を説明しやすい。
「物理本より安い」という事実が価格への納得感を生む。

### 計算式

```
PDF最低価格 = floor(物理本価格 × 0.50 / 100) × 100  （100円切り捨て）
PDF最高価格 = floor(物理本価格 × 0.70 / 100) × 100  （100円切り捨て）
```

**50〜70%の根拠**: 技術書典・BOOTHの観測値。
- 50%未満: PDF単体での価値を著しく低く見せてしまう
- 70%超: 物理本との差が小さく「PDF版を買う動機」が薄れる
- 同価格: 一部のサークルが採用するが、物理本の販売機会を喰う可能性がある

### 物理本と同時頒布時の推奨価格

物理本1,000円の場合: PDF 500〜700円
- 500円: 気軽に買える価格。フォロワー獲得重視の場合
- 700円: 収益重視。「物理本より300円安い」という訴求ができる
- 無料配布: 次回作への誘導・認知拡大目的。収益は捨てる判断

## 4. 市場相場データ

### データソース（2025年調査）

- 技術書典16 サークル報告（Zenn・note 複数記事）
- BOOTH 同人誌価格分布
- 「この程度の冊子が1000円」議論（湊川あい note）

### 60ページ前後の実態

| 価格帯 | 比率 | 備考 |
|-------|------|------|
| 500円以下 | 少数 | 無料〜薄い本 |
| 600〜800円 | 一定数 | 印刷費ベース計算 |
| **1,000円** | **最多** | 技術書典の暗黙の標準 |
| 1,200〜1,500円 | 少数 | 分厚い・カラー本 |

**1,000円均一**が慣例化している背景: ページ数より「欲しい情報があるか」で買う文化が定着しているため、
細かな価格差より「1,000円という分かりやすさ」が優先される。

### 完全受注生産の価格慣例

受注生産であることを理由に価格を上乗せするサークルはほぼ存在しない。
技術書典の公式システムが「電子版+受注生産」形式をサポートしており、
購買者側もその仕組みを理解している。

## 5. アルゴリズム更新の判断基準

以下のケースでテーブルや計算式を見直す。

| 更新トリガー | 対応箇所 |
|------------|---------|
| 日光企画の価格改定 | `NIKKO_ONDEMAND_A5_MONO` テーブル・データ取得日 |
| 他の印刷所を使う | テーブルを差し替え（または追加） |
| 市場相場が変わった | `MARKET_PHYSICAL_STANDARD` と市場相場セクション |
| PDF比率の慣例が変わった | `_pdf_range()` の係数（0.50〜0.70） |

更新時は `book.yaml` の `quote_date` も更新すること（90日経過で警告が出る）。
