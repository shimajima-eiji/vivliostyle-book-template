# vivliostyle-book-template

[日本語](./README.ja.md)

Write your book in Markdown, get a print-ready PDF. A self-publishing
template built on [Vivliostyle](https://vivliostyle.org/) (CSS typesetting),
designed so the manuscript stays plain Markdown and everything else — index,
cover, spine, print imposition — is generated by `make`.

The same template also drives a [Re:VIEW](https://reviewml.org/) build: drop in
a `book/config.yml` and the engine switches formats automatically. So you can
start in Markdown and graduate to Re:VIEW without changing your toolchain.

Built for indie authors and technical self-publishers who want their source to
stay in Git, not in a proprietary DTP app.

## Why this template

Most self-publishing toolchains force a trade-off: either you write in a WYSIWYG
editor (no diffs, no Git, no automation) or you fight LaTeX for months. This
template takes a third path — **your manuscript is Markdown, and CSS does the
typesetting**. You get clean diffs, scriptable builds, and a PDF that's ready
for a print shop.

- **Manuscript stays Markdown.** Chapters live in `manuscript/`. No binary blobs.
- **One command to build.** `make build` runs CSS → index → chapters → typeset → cover.
- **Cover, spine and back are generated**, not hand-placed, so they stay in sync.
- **Print-ready output is assembled for you** — interior, cover, back cover and a
  "net print" PDF — in A5 and B5, via `make ship`.
- **Two formats, one workflow.** Vivliostyle (CSS) by default; Re:VIEW (LaTeX) when
  you add `book/config.yml`. The engine auto-detects which one you're using.

## Requirements

| Tool | Used for |
|------|----------|
| [mise](https://mise.jdx.dev/) | Node.js version management (pins Node 24) |
| [uv](https://docs.astral.sh/uv/) | Python runtime for cover / index / build scripts |
| ImageMagick (`magick`) | Resizing and merging cover artwork |
| poppler / qpdf | `pdfinfo`, `qpdf` — page-count checks and PDF assembly |

```bash
# macOS
brew install mise uv imagemagick poppler qpdf
mise install   # installs Node 24 as pinned in mise.toml
```

> Re:VIEW builds additionally require a Ruby + Re:VIEW + TeX Live toolchain
> (`bundle exec review-pdfmaker`). The Vivliostyle path needs none of that.

## Setup

This repo is a **template you copy from**, not one you edit in place. Keep it
next to your book repo so your book can reference `Makefile.engine` and the
shared cover/index scripts:

```bash
# 1. Clone the template as "book-template"
git clone https://github.com/shimajima-eiji/vivliostyle-book-template book-template

# 2. Create your own book repo alongside it
mkdir my-book && cd my-book
git init

# 3. Copy the template files in and initialize
cp -r ../book-template/. .
bash scripts/init.sh
```

`scripts/init.sh` asks for a title and author, fills in the placeholders, and
rewrites your `Makefile` so it includes `../book-template/Makefile.engine`.
Resulting layout:

```
workspace/
  book-template/   # this repo — leave it untouched
  my-book/         # your book repo
```

## Writing

1. Define your chapter entries in `vivliostyle.config.js`.
2. Start writing in `manuscript/ch01/main.md`.
3. Build:

```bash
make build         # full build: CSS -> index -> chapters -> typeset -> cover
make vivliostyle   # typeset only (fast preview while writing) -> dist/book-digital.pdf
make index         # regenerate the index only
```

`make vivliostyle` produces `dist/book-digital.pdf`. The print-ready bundles
(interior + cover + net print, per paper size) come from `make ship` — see
below.

## Book configuration

Per-book settings live in `book.yaml`:

```yaml
title: "Your Title"
author: "Your Name"

cover:
  title_line1: "Line 1 of the title"
  title_line2: ""              # leave empty if unused
  subtitle_line1: ""
  subtitle_line2: ""
  font_size_title: 110
  bg_image: "cover/_fixed/front-source.png"  # canonical source image

css:
  chapter_image: false   # set true to use per-chapter title images

print:
  binding: "wireless"    # "wireless" = perfect binding (multiples of 2)
                         # "saddle"   = saddle stitch (multiples of 4)
```

## Cover, spine and chapter images

Refresh all cover artwork with `make cover-assets` (it runs the front / back /
spine / preview / print steps). To customize generation, place these scripts in
your book repo — each one calls the template's shared base generator:

| File | Generates |
|------|-----------|
| `cover/cover-data.py`       | front cover → `cover/cover-book.png` |
| `cover/back-cover-data.py`  | back cover  → `cover/back-book.png` |
| `cover/spine-data.py`       | spine       → `cover/spine-book.png` |
| `assets/chapter-data.py`    | chapter title images → `assets/chapter-XX.png` |

Each script imports the template's base generator (`init.sh` scaffolds
`assets/chapter-data.py` for you):

```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).parent.parent.parent / "book-template" / "cover"))
from gen_cover_base import generate
```

## Building print-ready PDFs

`make ship` builds the book and assembles everything a print shop needs, laid
out under `dist/<SIZE>/`:

```bash
make ship      # build A5 and B5 (default)
make ship-a5   # A5 only
make ship-b5   # B5 only
```

Output layout per size (e.g. A5):

```
dist/A5/
  電子版.pdf            # digital edition: front cover + interior + back cover
  製本版/               # print edition
    pdf/  本文.pdf 表紙.pdf 裏表紙.pdf ネットプリント.pdf
    raw/  表紙.png 裏表紙.png 背表紙.png
```

`ship` also commits and pushes the result (it's built to run in a Git repo).
For B5, generate the B5 cover masters once with `make cover-b5-prep` before the
first `make ship-b5`.

> The labels are Japanese by convention: `電子版` = digital edition,
> `製本版` = print edition, `本文` = interior, `表紙`/`裏表紙`/`背表紙` =
> front/back/spine, `ネットプリント` = a single PDF padded with blank pages for
> walk-in / convenience-store printing.

## Validating the output

```bash
make preflight   # check every PDF in dist/ (page count, blank pages, binding multiple)
make spread      # render two-page spread previews
```

`preflight` reads `print.binding` from `book.yaml` and verifies the page count
is a valid multiple (2 for perfect binding, 4 for saddle stitch) so the book
folds cleanly.

## Re:VIEW format (optional)

If you'd rather typeset with Re:VIEW / LaTeX, convert your Markdown and let the
engine switch formats automatically — it detects Re:VIEW when `book/config.yml`
exists:

```bash
make convert     # convert manuscript/*.md to Re:VIEW (.re) under book/
make ship        # same ship pipeline, now via Re:VIEW
```

`make convert` does the structural conversion only; index entries, footnotes and
columns are added afterwards (the project's CLAUDE.md documents the LLM
post-processing pass).

## Cover asset conventions

Keep "adopted", "candidate", "corrected" and "scratch" artwork separate so a
build never picks up the wrong file:

```text
cover/_work/        per-book scratch workspace (gitignored)
  incoming/         freshly received files
  scratch/          intermediate conversions / experiments
  compare/          comparison renders
  preview/          local-only preview output
cover/_candidates/  generated / external candidates (tracked)
cover/_fixed/       corrected masters & adopted sources (tracked)
cover/fix-ledger.md ledger: which fixed input is currently adopted
cover/              only the canonical files the build reads
cover/print/        print-resolution output (A5); cover/print-b5/ for B5
```

Rules:

- `cover/cover-book.png`, `cover/back-book.png`, `cover/spine-book.png` are the
  canonical files the build consumes.
- Record the currently adopted source in `cover/fix-ledger.md`.
- Don't create ambiguous names like `copy` or `final2`.
- Don't drop loose files directly under `_work/`.
